# Kraken Team Setup Wizard

Configure your personal MySQL credentials for the Integration Hub database. Each PM has their own username and password.

## Step 1 ŌĆö Check Existing Config

Check if a `.env` file already exists in the current working directory.

**If it exists**, read it and check for `KRAKEN_MYSQL_USER` and `KRAKEN_MYSQL_PASSWORD`:
- If both are set and non-empty:
  > "You already have credentials configured:
  > - User: [username]
  > - Password: ****
  >
  > Do you want to update them? (yes/no)"
  - If no ŌåÆ skip to Step 4 (verify)
  - If yes ŌåÆ proceed to Step 2

**If it doesn't exist**, proceed to Step 2.

## Step 2 ŌĆö Collect Credentials

Ask the PM:
> "Enter your MySQL credentials for the Integration Hub database (medusa-db).
> If you don't have credentials yet, ask your team lead."

Ask for:
1. **Username**: "What is your MySQL username?"
2. **Password**: "What is your MySQL password?"

## Step 3 ŌĆö Write .env File

Write the credentials to a `.env` file in the current working directory:

```
# Kraken Team MySQL credentials (integration-hub-db)
# Generated by /setup-kraken
export KRAKEN_MYSQL_USER=<username>
export KRAKEN_MYSQL_PASSWORD=<password>
```

**Important:** Check if `.gitignore` exists in the project root. If it does, check if `.env` is listed. If `.env` is NOT in `.gitignore`, warn the PM:

> "Warning: `.env` is not in your `.gitignore`. Your credentials could be committed to git. Add `.env` to your `.gitignore` file."

If `.gitignore` does not exist, create one with `.env` as the only entry and inform the PM.

## Step 4 ŌĆö Verify Connectivity

Attempt to connect to the database using the credentials:

```bash
python3 -c "
import pymysql
conn = pymysql.connect(
    host='medusa-db.cluster-ro-cwxvzubmfq3d.eu-west-1.rds.amazonaws.com',
    user='<username>',
    password='<password>',
    database='integration-hub-db',
    port=3306,
    connect_timeout=10
)
cursor = conn.cursor()
cursor.execute('SELECT COUNT(*) FROM integrations')
count = cursor.fetchone()[0]
conn.close()
print(f'OK:{count}')
"
```

**If connection succeeds:**
> "Connection verified. Found [count] integrations in the database."

**If connection fails:**
> "Connection failed: [error message]. Please check your credentials and try again."
> Re-run Step 2.
